Full spec and constraints in `notes/02-scaffold-vite-react-typescript-project.md`. Task ends when `npm run dev` starts a blank page with no console errors. [28m 12s]

- [x] **Configure TailwindCSS** — Install `tailwindcss@3 postcss autoprefixer` as dev deps (pin to v3 — bare `tailwindcss` resolves v4 which has incompatible setup). Run `npx tailwindcss init -p`. Set `content: ["./index.html", "./src/**/*.{ts,tsx}"]` in `tailwind.config.js`. Add three separate directives to the top of `src/index.css`: `@tailwind base;`, `@tailwind components;`, `@tailwind utilities;`. Replace `index.html` title with `Mind`. Full setup spec and v3 vs v4 gotchas: `notes/03-tailwindcss-v3-setup.md`. Task ends when a `className="text-blue-500"` element renders in the correct color. [9m 56s]

- [x] **Configure project structure and environment** — Create directories: `src/core/api/`, `src/core/auth/`, `src/core/types/`, `src/pages/`, `src/components/`. Create `.env.local` (gitignored) with `VITE_API_BASE_URL=http://localhost:3002`. Create `.env.example` with the same key and an empty value. Create `src/core/config.ts` exporting `const API_BASE_URL = import.meta.env.VITE_API_BASE_URL as string` with a startup assertion that it is non-empty. Configure `@` path alias: add `resolve.alias` to `vite.config.ts` (requires `npm install -D @types/node` for `import { resolve } from 'path'`); add `paths` and `baseUrl` to `tsconfig.app.json` (not the root `tsconfig.json` — the modern Vite template's app code compiles under `tsconfig.app.json`). Task ends when `import { API_BASE_URL } from '@/core/config'` resolves without errors. [6m 57s]

- [x] **Configure React Router and page shells** — Create `src/router.tsx` with `createBrowserRouter`. Routes: `/login` → `LoginPage` (public), `/deeplink-auth` → `MagicLinkPage` (public), `/auth/google/callback` → `GoogleCallbackPage` (public), `/sessions` → `SessionsPage` (protected), `/sessions/:id` → `SessionsPage` with selected session (same component, protected), `/calibrations` → `CalibrationPage` (protected), `/` → redirect to `/sessions`. `ProtectedRoute` redirects to `/login` when token absent. Create stub components for each page. Wrap `src/main.tsx` with `RouterProvider`, `QueryClientProvider`, `AuthProvider`. Task ends when navigating to each URL renders the correct stub. [7m 34s]

## Phase 2 — Authentication

- [x] **Auth context + token management** — Create `src/core/auth/AuthContext.tsx`. Context value: `{ token: string | null; login(token: string): void; logout(): void; pendingEmail: string | null; setPendingEmail(email: string): void; clearPendingEmail(): void }`. `token` persisted under `mind_auth_token`; `pendingEmail` under `mind_pending_email` — both in `localStorage`, both managed exclusively by `AuthContext` (no other file may read or write these keys). `login()` stores token and navigates to `/sessions`. `logout()` clears token and navigates to `/login`. Export `useAuth()` hook. `ProtectedRoute` reads from `useAuth().token`. Task ends when `useAuth().logout()` from a protected page redirects to `/login`. [13m 24s]

- [x] **Login page: email OTP flow** — Implement `LoginPage`. Step 1: email input + "Send code" button → `POST /auth/send-code { email }`. On success: call `auth.setPendingEmail(email)` (persists under `mind_pending_email` via AuthContext — do not write to localStorage directly), advance to step 2. Step 2: 6-digit code input + "Verify" button → `POST /auth/verify-code { email, code }` returns `{ accessToken, user }` → `auth.login(accessToken)`. Show loading spinner during requests. Show inline error on failure (extract `message` field from response body). Centered card, max-width 400px. Note: these REST endpoints live in mind_api Phase 21. [9m 12s]

- [x] **Magic link auto-login** — Implement `MagicLinkPage` at `/deeplink-auth`. The API email template already contains `{APP_BASE_URL}/deeplink-auth?code=XXXXXX` — on mobile this opens as a native deeplink, in a browser this hits this route. On mount: read `?code=` from URL query params. Read `auth.pendingEmail` (set by `LoginPage` via `auth.setPendingEmail()`). If both present: call `POST /auth/verify-code { email, code }` → `auth.login(token)` → navigate to `/sessions`. If code is present but `auth.pendingEmail` is null: show a short email input so the user can confirm their address, then verify. Call `auth.clearPendingEmail()` after successful login. No changes to API email templates needed — the existing deeplink URL works for web as-is. [5m 58s]

- [x] **Login page: Google Sign-In** — Add "Continue with Google" button on `LoginPage`. On click: call `GET /auth/google` → API redirects browser to Google OAuth. After Google auth, API relays to `{WEB_BASE_URL}/auth/google/callback?googleCode=<code>` (success) or `?googleError=<err>` (failure). Implement `GoogleCallbackPage`: on `googleCode` → call `POST /auth/google { code, redirectUri: window.location.origin + '/auth/google/callback' }` → receive `{ accessToken, user }` → `auth.login(accessToken)`. On `googleError` → redirect to `/login?error=google`. Note: endpoint spec in mind_api Phase 21 (`.ai-factory/ROADMAP.md`). [14m 24s]

- [x] **API client with auth interceptor** — Create `src/core/api/client.ts`. `apiFetch<T>(path, options?)`: prepend `API_BASE_URL`, add `Content-Type: application/json` and `Authorization: Bearer <token>` headers. On 401: clear token, navigate to `/login`. On non-2xx: throw `ApiError(status, message)` with message extracted from response JSON. Task ends with the client used by at least one page. [6m 54s]

## Phase 3 — Sessions Split-Panel Page

Single page at `/sessions` (and `/sessions/:id` for selected state). Left column: session list. Right panel: session charts. Selecting a session updates the URL to `/sessions/:id` and populates the right panel — no page navigation.

- [x] **Split-panel layout shell** — Implement `SessionsPage` with a two-column layout: left column fixed-width (~280px), right panel fills remaining width. Left column header: "Sessions" title + logout button. Right panel: empty state ("Select a session") when no session selected. Use TailwindCSS `flex h-screen overflow-hidden`. Left column is scrollable independently. Right panel is scrollable independently. [3m 30s]

- [x] **Session list (left column)** — Fetch `GET /sessions/runs` via React Query. Render a scrollable list of session rows: date (`DD MMM, HH:mm`) + duration (`mm:ss`). Selected session highlighted. Clicking a row navigates to `/sessions/:id` (via React Router, no reload). "Load more" at the bottom loads next page. Skeleton loader while fetching. "No sessions yet" empty state. [6m 54s]

- [x] **Session charts panel (right panel) — ECharts multi-grid** — When `/sessions/:id` is active, right panel fetches `GET /sessions/runs/:id/instructions` and `GET /sessions/runs/:id/biometrics` in parallel (full session range: `?from=startedAt&to=endedAt`; no incremental loading for MVP). Renders a single ECharts instance with up to 4 stacked grids sharing a linked X-axis (seconds from session start): (1) instruction phases — colored bars for `inhale`/`hold`/`exhale`/`rest` from `breath_phase` instructions; (2) heart rate from `sampleType: "cardio"` → `data.heartRate`; (3) EEG bands from `sampleType: "nfb"` → `data.{delta,theta,alpha,smr,beta}`; (4) emotions from `sampleType: "emotions"` → `data.{attention,relaxation,cognitiveLoad,cognitiveControl,selfControl}`. Omit any grid whose sample type has zero data points. All X-axes linked via `dataZoom` and `axisPointer.link`. Panel header: formatted date + total duration. Full spec: color palette, ECharts option skeleton, data transforms, dynamic grid layout in `notes/04-echarts-session-charts.md`. [18m 31s]

## Phase 4 — NFB Calibration History Page

- [x] **`/calibrations` page** — Fetch `GET /nfb-calibrations` (returns `{ records, total }`); load more with `?offset=N` as the user scrolls down. Group records by `deviceSerial`. For each device: section header with serial + "valid / total" badge. One ECharts `LineChart` (X-axis: `calibratedAt` ordered chronologically, Y-axis: value) with two lines: `individualFrequency` (Hz, alpha frequency) and `individualPeakFrequencyPower` (alpha peak power) — these values shift with each new user connection and the chart shows their evolution over calibration history. Invalid calibrations: hollow red dots. Valid: filled green dots. Tooltip shows `calibratedAt`, `isValid`, `failReason`. Empty state: "No calibrations recorded yet". [12m 3s]

## Phase 5 — Bug Fixes & Cleanup

- [x] **Fix Google OAuth callback parameter names** — `GoogleCallbackPage` reads `?code=` and `?error=` but the API relays `?googleCode=` and `?googleError=`; Google sign-in is completely broken. Two-line fix in `src/pages/GoogleCallbackPage/index.tsx`: `searchParams.get('code')` → `'googleCode'`, `searchParams.get('error')` → `'googleError'`. Do not change `LoginPage`'s `?error=google` round-trip or `MagicLinkPage`'s `?code=` — those are correct. Full spec: `notes/05-google-callback-fix.md`. [4m 13s]

- [x] **Fix auth state cleanup on logout and 401** — Two bugs: (1) `logout()` in `AuthContext` does not clear `mind_pending_email`, leaving a stale email that silently mismatch-verifies on the next magic-link visit; (2) `client.ts` throws `ApiError` after `window.location.assign('/login')`, racing callers' catch handlers with the hard navigation. Fix (1): inline `localStorage.removeItem(PENDING_EMAIL_KEY)` and `setPendingEmailState(null)` in `logout`. Fix (2): return `new Promise<T>(() => {})` after the `window.location.assign` call so no catch handler fires. Full spec: `notes/06-auth-state-cleanup.md`. [4m 41s]

- [x] **Optimize chart transforms and simplify chartOption** — Four independent improvements surfaced by code review: (1) Pre-partition the biometrics array by `sampleType` once (one `Map` pass) and hoist `startMs = new Date(startedAt).getTime()` before calling `toSeries` — eliminates 11× full-array scans and O(n×11) `Date` constructions; update `toSeries` signature to accept pre-filtered samples and `startMs: number`. (2) Wrap the `buildSessionChartOption` call in `SessionCharts.tsx` with `useMemo`. (3) Remove `orderMap` and the dead `.sort()` from `groupByDevice` in `CalibrationPage/transforms.ts` — `Map` iteration is already insertion-ordered. (4) Replace `allXAxisIndices` array in `chartOption.ts` with the ECharts built-in string `'all'`. Full spec: `notes/07-chart-performance-and-simplification.md`. [7m 14s]

- [ ] **Shared format utility and query co-location** — Three cleanup items: (1) Create `src/core/format.ts` with `formatDate` and `formatDuration`; delete the duplicate `format.ts` files in `SessionsPage/` and `CalibrationPage/`; update 4 import sites. (2) Extract a shared `<PageHeader title={...} />` component in `src/components/` from the identical header block copied in both `SessionsPage` and `CalibrationPage`. (3) Move the instructions and biometrics `useQuery` calls from `SessionsPage/index.tsx` into `SessionCharts.tsx` — they are the chart panel's data concern; the current location couples chart-data fetches to `SessionList` re-renders. `SessionChartsProps` shrinks to `{ session: SessionRun }`. Full spec: `notes/08-shared-format-and-query-location.md`.
