# Plan: Scaffold Vite + React + TypeScript project

## Context

Bootstrap the empty `mind_web/` directory with a working Vite + React + TypeScript scaffold, install runtime dependencies (`react-router-dom`, `@tanstack/react-query`, `echarts`, `echarts-for-react`), and strip generated boilerplate so `npm run dev` serves a blank page with no console errors. Full spec lives in `.ai-factory/notes/02-scaffold-vite-react-typescript-project.md` — this plan implements that note plus the v1 plan-review fixes (deterministic blank baseline, lint/build verification, `typecheck` script, pre-existing `node_modules`/lockfile reset).

## Settings
- Testing: no
- Logging: minimal
- Docs: no

## Assumptions
- The current `mind_web/` directory contains project metadata (`CLAUDE.md`, `AGENTS.md`, `.ai-factory/`, `.claude/`, `.mcp.json`, `.idea/`, `.gitignore`, empty `src/assets/`, empty `public/`) — all of which must be preserved.
- A leftover `node_modules/` (from an earlier scaffold attempt) may also be present; if `package.json` / `package-lock.json` exist they are stale and must be removed before scaffolding (Task 2 preamble).
- The Vite scaffolder will be run interactively (a TTY is available) — it cannot be backgrounded.
- `nvm` (or equivalent) is available locally if a Node version switch is required.

## Tasks

### Phase 1: Preflight

- [x] **Task 1: Verify Node version**
  Files: _(none — shell only)_
  Run `node --version` and confirm it satisfies `^20.19.0 || >=22.12.0`. If lower (including 20.0–20.18), switch with `nvm use 22` before proceeding. `create-vite@9` fails with `EBADENGINE` and a `SyntaxError` from `node:util`'s `styleText` export on older Node versions. Do not skip this check.

- [x] **Task 2: Reset stale scaffold artifacts** (depends on Task 1)
  Files: delete (if present) `mind_web/package.json`, `mind_web/package-lock.json`, `mind_web/node_modules/`
  Check whether `package.json` or `package-lock.json` already exists at the repo root. If either is present, delete both **and** delete `node_modules/` so Task 5 reinstalls cleanly against the Vite-generated `package.json`. **Do not** touch `CLAUDE.md`, `AGENTS.md`, `.ai-factory/`, `.claude/`, `.mcp.json`, `.idea/`, `.gitignore`, `src/`, or `public/` — only the three scaffold artifacts above. If none of the three exist, this task is a no-op.

### Phase 2: Scaffold

- [x] **Task 3: Run Vite scaffolder interactively** (depends on Task 2)
  Files: generates `package.json`, `vite.config.ts`, `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`, `eslint.config.js`, `index.html`, `src/App.tsx`, `src/App.css`, `src/main.tsx`, `src/index.css`, `src/vite-env.d.ts`, `src/assets/react.svg`, `public/vite.svg`
  From inside `mind_web/`, run **interactively** (do not background, do not pipe input):
  ```
  npm create vite@latest . -- --template react-ts
  ```
  When Vite warns "Current directory is not empty…", choose **"Ignore files and continue"**.
  - ❌ Do NOT select "Remove existing files and continue".
  - ❌ Do NOT pass `--overwrite` (wipes `.ai-factory/`, `CLAUDE.md`, `AGENTS.md`, `.claude/`, `.mcp.json`, `.idea/`).
  - ❌ Do NOT pass `--force` (not a valid `create-vite` flag — silently ignored by `npm create`).
  With "Ignore files and continue", every pre-existing file is left untouched **by the scaffolder** (including `.gitignore`); reconciliation happens in Task 4.

- [x] **Task 4: Reconcile `.gitignore`** (depends on Task 3)
  Files: `.gitignore`
  Diff Vite's default `.gitignore` against the existing one and append only the missing lines. Specifically check for and add (if absent): `dist-ssr`, `*.local`, `*.tsbuildinfo`, `node_modules/.cache`. Existing entries (`node_modules/`, `dist/`, `.vite/`, `*.log`, `.idea/`, `.vscode/`, `.env.local`, `.env.*.local`) must remain — do not reorder, do not deduplicate against broader patterns (e.g. keep `.env.local` even though `*.local` covers it).

### Phase 3: Configure & install

- [x] **Task 5: Update `package.json` metadata and scripts** (depends on Task 3)
  Files: `package.json`
  - Set `"name": "mind-web"` regardless of what Vite generated (Vite's slugification of `mind_web` is not guaranteed across versions).
  - Add top-level `"engines": { "node": ">=20" }`.
  - Add `"typecheck": "tsc --noEmit"` to `scripts` (referenced by `mind_web/CLAUDE.md`; not generated by the Vite template).
  - Leave all other Vite-generated fields untouched: `"private"`, `"type"`, `"version"`, existing `scripts` (`dev`, `build`, `lint`, `preview`), `"dependencies"`, `"devDependencies"`.

- [x] **Task 6: Install template dependencies + runtime libs** (depends on Task 5)
  Files: `package.json`, `package-lock.json`, `node_modules/`
  1. Run `npm install` (no args) to resolve template dependencies and create `package-lock.json`.
  2. Run `npm install react-router-dom @tanstack/react-query echarts echarts-for-react` to add the runtime libraries listed in the milestone.

- [x] **Task 7: Verify dev dependencies (no new installs)** (depends on Task 6)
  Files: `package.json`, `package-lock.json`
  Run `npm install` again — it should be a no-op. Confirm `@types/react`, `@types/react-dom`, and `eslint` are present under `devDependencies` and locked in `package-lock.json`. **Do NOT install** `@typescript-eslint/eslint-plugin` or `@typescript-eslint/parser` — the `react-ts` template uses the unified `typescript-eslint` package via the flat config in `eslint.config.js`. The ROADMAP milestone text mentions the deprecated split packages; that text is wrong and is superseded by `notes/02-scaffold-vite-react-typescript-project.md` constraint #2.

### Phase 4: Strip boilerplate & verify

- [x] **Task 8: Strip generated boilerplate** (depends on Task 6)
  Files: delete `src/App.css`, delete `src/assets/` (entire directory, including the new `react.svg`), delete `public/vite.svg`; edit `src/App.tsx`; empty `src/index.css`
  Replace the **entire contents** of `src/App.tsx` with:
  ```tsx
  function App() {
    return <div />;
  }
  export default App;
  ```
  This must remove every default import — `import './App.css'`, `import reactLogo from './assets/react.svg'`, `import viteLogo from '/vite.svg'`, **and `import { useState } from 'react'`** — as well as the `count` state and the full JSX block. Leaving `useState` behind triggers an unused-import lint warning that violates the "no warnings" acceptance criterion.
  Also **truncate `src/index.css` to an empty file**. The default Vite `index.css` sets `background-color: #242424` and `color-scheme: light dark`, which renders dark gray on dark-mode systems and breaks the "blank white page" acceptance check. The next milestone (TailwindCSS setup) will overwrite this file with `@tailwind` directives anyway.
  Leave `src/main.tsx`, `src/vite-env.d.ts`, `index.html`, `eslint.config.js`, `vite.config.ts`, and all `tsconfig*.json` files untouched.

- [x] **Task 9: Verify lint, typecheck, build, and dev server** (depends on Task 8)
  Files: _(none — runtime checks)_
  Run all four in order and require each to pass cleanly with no errors and no warnings from application code:
  1. `npm run lint` — must complete with zero errors and zero warnings. This is the authoritative check that no stray imports remain in `App.tsx` (Vite's dev server does not run ESLint, so the browser console alone cannot detect this).
  2. `npm run typecheck` — must complete with zero errors.
  3. `npm run build` — must complete and emit `dist/`.
  4. `npm run dev` — confirm:
     - Vite starts on `http://localhost:5173` (auto-bump to `5174` / `5175` is acceptable if the port is busy).
     - The browser renders a blank page (white on light-mode systems; with `index.css` emptied in Task 8 the page should be the user-agent default — no `#242424` dark fill from the template).
     - DevTools console shows no errors and no warnings from application code (Vite HMR connection messages and the React DevTools install hint are acceptable).
  Stop the dev server when done. If any check fails, return to Task 8 and re-verify `App.tsx` imports and `index.css` contents.
